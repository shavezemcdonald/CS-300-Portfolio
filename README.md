# CS-300 Portfolio

## Overview
This portfolio includes my runtime analysis project, my ABCU advising program implementation, and reflections showing growth in data structures and algorithm design.

## Portfolio Reflection

1. What was the problem you were solving?
I was solving how to efficiently store, organize, sort, and retrieve course information for the ABCU advising system while evaluating which data structures performed best through runtime and memory analysis.

2. How did you approach the problem?
I compared vectors, hash tables, and binary search trees using Big O analysis, then selected and implemented a structure that balanced efficiency, maintainability, and performance. This project strengthened my understanding of why data structures matter in software development.

3. How did you overcome roadblocks?
I worked through issues with loading data, debugging logic, refining pseudocode into working C++ code, and validating output through repeated testing. Each challenge improved my troubleshooting and problem solving.

4. How has this project expanded your approach to designing software?
This work taught me to focus more on planning, algorithm efficiency, modular design, and selecting the right data structure before coding. I now approach software development more strategically.

5. How has your work evolved in maintainability and adaptability?
My programming has become more organized, readable, adaptable, and performance conscious. I now place greater emphasis on documentation, reusable logic, and writing code that can scale.
